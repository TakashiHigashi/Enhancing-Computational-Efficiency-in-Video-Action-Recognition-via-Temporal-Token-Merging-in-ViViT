# リポジトリ構成サマリー

本ドキュメントでは、本リポジトリに含まれるすべてのファイルとディレクトリの概要を説明します。

## 📁 ルートディレクトリ

| ファイル/ディレクトリ | 説明 |
|-------------------|------|
| `README.md` | リポジトリの概要、目次、使い方の総合ガイド |
| `研究業績まとめ.md` | 研究成果の詳細な総括文書 |
| `使い方ガイド.md` | リポジトリの使い方とベストプラクティス |
| `チェックリスト.md` | 研究活動の進捗確認用チェックリスト |
| `CONTRIBUTING.md` | 貢献方法とコーディング規約 |
| `.gitignore` | Gitで追跡しないファイルの設定 |

## 📂 ディレクトリ詳細

### `experiments/` - 実験コード

研究で実施した実験のコードとスクリプトを格納するディレクトリ。

- `README.md`: 実験の概要と一覧
- `common/experiment_template.md`: 新しい実験を追加する際のテンプレート
- 各実験ごとにサブディレクトリを作成

**構造例:**
```
experiments/
├── README.md
├── common/
│   └── experiment_template.md
└── experiment1/
    ├── README.md
    ├── data/
    ├── scripts/
    └── results/
```

### `tools/` - ツール・ライブラリ

研究過程で開発した再利用可能なツールやライブラリ。

- `README.md`: ツールの一覧と使用方法
- `templates/tool_template.md`: 新しいツールのドキュメントテンプレート
- 各ツールごとにサブディレクトリを作成

**構造例:**
```
tools/
├── README.md
├── templates/
│   └── tool_template.md
└── tool1/
    ├── README.md
    ├── src/
    └── tests/
```

### `docs/` - 技術文書

設計書、API仕様書、研究ノートなどの技術文書。

- `README.md`: 文書の一覧と構成
- `templates/document_template.md`: 新しい文書のテンプレート

**格納する文書の例:**
- システム設計書
- アーキテクチャ設計書
- API仕様書
- 環境構築ガイド
- 研究ログ

### `data/` - データセット

研究で使用・作成したデータセットを格納。

- `README.md`: データセットの一覧とメタデータ

**注意事項:**
- 大容量ファイルはGit LFSを使用
- 機密データは`.gitignore`に追加
- 各データセットにメタデータを記録

### `publications/` - 論文・発表資料

学術論文、学会発表資料、ポスターなどの成果物。

- `README.md`: 論文・発表の一覧と詳細
- `papers/`: 論文PDF
- `slides/`: 発表スライド
- `posters/`: ポスター
- `bibtex/`: BibTeX情報

**構造例:**
```
publications/
├── README.md
├── papers/
│   ├── conference/
│   ├── domestic/
│   └── journal/
├── slides/
│   ├── oral/
│   └── invited/
├── posters/
└── bibtex/
```

## 🔄 ファイル追加のワークフロー

### 新しい実験を追加する場合

1. `experiments/`に新しいディレクトリを作成
2. `experiments/common/experiment_template.md`をコピーして編集
3. 実験コードとデータを追加
4. `experiments/README.md`を更新

### 新しいツールを追加する場合

1. `tools/`に新しいディレクトリを作成
2. `tools/templates/tool_template.md`をコピーして編集
3. ツールのコードとテストを追加
4. `tools/README.md`を更新

### 論文・発表を追加する場合

1. PDFを適切なディレクトリに配置
2. `publications/README.md`を更新
3. `研究業績まとめ.md`も更新

## 📊 ベストプラクティス

1. **一貫性**: テンプレートを使用して一貫した構造を維持
2. **文書化**: すべてのディレクトリにREADME.mdを配置
3. **バージョン管理**: 重要な変更は明確なコミットメッセージで記録
4. **整理整頓**: 定期的にファイルを整理し、不要なものを削除

## 🎯 活用方法

- **研究進捗管理**: チェックリストで進捗を追跡
- **成果のまとめ**: 研究業績まとめで成果を可視化
- **ポートフォリオ**: 就職活動でのアピール材料として
- **知識の蓄積**: 実験やツールを体系的に整理
